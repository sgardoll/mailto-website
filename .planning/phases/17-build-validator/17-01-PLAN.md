Create apps/workflow_engine/validator.py. Use only stdlib: `re`, `html.parser`. No new deps.

Module structure:

```
"""Validate LM-generated Alpine/Tailwind HTML modules."""
from __future__ import annotations

import re
from html.parser import HTMLParser, HTMLParseError
from .logging_setup import get

log = get("validator")
```

Implement `_Inspector(HTMLParser)` subclass that accumulates in a single parse pass:
- `has_x_data: bool` — set True when any tag has attribute "x-data"
- `has_event_handler: bool` — set True when any tag has @click, @input, @change, or any key starting with "x-on:"
- `x_if_on_div: bool` — set True when tag=="div" and "x-if" in attr_dict
- `cdn_src_urls: list[str]` — collect src attribute of every `<script>` tag
- `script_texts: list[str]` — accumulate character data inside `<script>` tags
- `_in_script: bool` and `_script_buf: list[str]` — internal state for script text collection

Implement `validate_module(html: str) -> list[str]`. Checks in this order:

1. VAL-06: `if len(html.encode("utf-8")) < 800:` → append "HTML shorter than 800 bytes"
2. VAL-03 (non-ellipsis stubs, full HTML): compile case-insensitive regex for:
   `r"\bTODO\b"`, `r"\bFIXME\b"`, `r"\bplaceholder\b"`, `r"coming soon"`, `r"//\s*implement"`
   — if any match on full html string → append stub error
3. Parse: `inspector = _Inspector(); inspector.feed(html)` — wrap in try/except HTMLParseError
   - On HTMLParseError: append "HTML not parseable: {e}" and return early
4. VAL-01 x-data: `if not inspector.has_x_data:` → append "Missing x-data attribute"
5. VAL-01 event handler: `if not inspector.has_event_handler:` → append "Missing event handler (@click/@input/@change/x-on:*)"
6. VAL-04: `if inspector.x_if_on_div:` → append "x-if used on <div> element (must be on <template>)"
7. VAL-02 Alpine CDN: check `any(re.search(r"cdn\.jsdelivr\.net/npm/alpinejs@[\d.]+/", u) for u in inspector.cdn_src_urls)`
   — if False → append "Missing pinned Alpine.js CDN <script> tag"
8. VAL-02 Tailwind CDN: check `any("cdn.tailwindcss.com" in u for u in inspector.cdn_src_urls)`
   — if False → append "Missing Tailwind CDN <script> tag"
9. VAL-03 ellipsis (script text only): `for t in inspector.script_texts: if re.search(r"\.\.\.", t):` → append "Ellipsis (...) found in script block"; break
10. VAL-05 external fetch/XHR: for each script text, use regex to extract fetch() URL arguments
    and XMLHttpRequest open() URL arguments; reject if URL does not start with "http://localhost"
    or "http://127.0.0.1" or "https://localhost" or "https://127.0.0.1".
    Pattern for fetch: `re.findall(r'fetch\s*\(\s*["\']([^"\']+)["\']', script_text)`
    Pattern for XHR: `re.findall(r'\.open\s*\(\s*["\'][^"\']*["\'\s]*,\s*["\']([^"\']+)["\']', script_text)`
    For each extracted URL, if not localhost/127.0.0.1 → append "External fetch/XHR to non-localhost: {url}"

CRITICAL PITFALLS to avoid (from RESEARCH.md):
- Pitfall 3: Tailwind URL has no version in path — check `cdn.tailwindcss.com` as literal, NOT with @version regex
- Pitfall 4: Ellipsis check only scans script_texts, not full HTML
- Pitfall 5: Both @click shorthand AND x-on:* full form must set has_event_handler=True
- Pitfall 6: localhost fetch must PASS (no error)
