Write five string constants. Each must be a complete, self-contained HTML document (~50-100 lines)
that passes validate_module() — meaning it must have:
- `<script src="https://cdn.jsdelivr.net/npm/alpinejs@3.14.1/dist/cdn.min.js" defer></script>`
- `<script src="https://cdn.tailwindcss.com"></script>`
- At least one element with `x-data="..."` (real Alpine data, not empty string)
- At least one event handler: `@click`, `@input`, `@change`, or `x-on:*`
- NO `x-if` on a `<div>` — use `<template x-if="...">` if conditional rendering is needed
- NO stub phrases (TODO, FIXME, placeholder, coming soon, // implement)
- NO ellipsis (...) in script blocks
- NO fetch() or XHR to external domains
- Total byte length >= 800 bytes (repeat body content or add comments if needed)
- `Alpine.store()` is only used inside `document.addEventListener('alpine:init', () => {...})`

Content guidelines per kind (content is Claude's discretion — make them realistic and useful):
- CALCULATOR_EXEMPLAR: A BMI or unit-conversion calculator. x-data holds input values and result.
  A button with @click triggers the formula. Result displayed with x-text.
- WIZARD_EXEMPLAR: A 2-step form wizard. x-data holds current step (step: 1) and form values.
  Back/Next buttons with @click="step--"/@click="step++" (or similar). Each step shown with
  x-show. Use `<template x-if="step === 2">` if needed (on template, NOT on div).
- DRILL_EXEMPLAR: A flashcard or quiz drill. x-data holds question, answer, revealed state.
  A "Reveal" button with @click="revealed = true". Answer shown/hidden via x-show="revealed".
- SCORER_EXEMPLAR: A rubric scorer with 2-3 sliders or number inputs. x-data holds dimension
  scores and computes weighted total. @input updates the score. Displays final score with x-text.
- GENERATOR_EXEMPLAR: A content generator with 1-2 text inputs and a Generate button.
  x-data holds the inputs and output. @click on Generate assembles output from template string.
  Displays result with x-text.
