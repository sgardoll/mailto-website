<objective>
Add the clean-shutdown lifecycle to the Flask server: POST /exit endpoint (with threading-based dispatch to avoid deadlock), atexit cleanup, SIGINT handler, and the Exit Setup button wired in index.html. Also complete the pre-flight error path so a non-writable directory surfaces a clear error to the user rather than a Python traceback.

Purpose: SRV-03 requires the process to terminate cleanly on both "Exit Setup" click and Ctrl-C. The critical pitfall here is calling server.shutdown() directly from inside a Flask request handler — this deadlocks. This plan implements the correct threading-based dispatch pattern. SRV-04 completes the pre-flight error surface.

Output: setup/server.py (amended), setup/templates/index.html (amended)

CRITICAL PITFALL: Do NOT call Flask's server shutdown directly from the /exit route handler. The pattern must be:
  threading.Thread(target=_shutdown_server, daemon=True).start()
  return jsonify({"ok": True})
where _shutdown_server() uses os.kill(os.getpid(), signal.SIGINT) to trigger a clean shutdown.

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Browser → POST /exit | Request comes from localhost browser only; no auth needed for localhost-only tool |
| /exit → OS signal | Uses os.kill(os.getpid(), SIGINT) — targets own PID only |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-01 | Denial of Service | POST /exit | accept | Localhost-only tool; any local user can kill the process anyway; no auth layer needed per requirements (out of scope) |
| T-03-02 | Tampering | _shutdown_server() threading | mitigate | Thread is daemon=True so it cannot block process exit; os.kill targets own PID — no injection vector |
| T-03-03 | Denial of Service | Deadlock via direct shutdown() | mitigate | _shutdown_server() uses os.kill(os.getpid(), SIGINT), not server.shutdown() from handler; threading.Thread wrapper ensures response returns before signal fires |
| T-03-04 | Information Disclosure | atexit _cleanup() message | accept | Prints only "Setup wizard exited cleanly." — no paths, credentials, or state |
</threat_model>
