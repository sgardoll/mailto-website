## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Email inbox config → pipeline router | pipeline_version determines code path (v1 vs v2) |
| Orchestrator → BUILD/INTEGRATE modules | Orchestrator invokes but does not validate the staged outputs (validator gate lives in build.py) |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-18-19 | Spoofing | `getattr(inbox, "pipeline_version", "v1")` fallback | mitigate | Unknown/missing pipeline_version safely defaults to v1 (least-privilege default); explicit "v2" required to enter new path |
| T-18-20 | Denial of Service | BUILD or INTEGRATE failure | mitigate | Explicit try/except around each; per-stage outcome recorded; orchestrator does not retry silently |
| T-18-21 | Information Disclosure | `_reply_failure` forwarding exception text | accept | Consistent with v1 behaviour; exception strings are engineer-crafted, not user-input-derived |
| T-18-22 | Tampering | `startup_assert_gitignore` bypass via .gitignore absent | accept | .gitignore absence is benign for a fresh site_dir; git_ops auditing still enforced per-commit. Check is a belt-and-braces check, not primary defense |
</threat_model>
