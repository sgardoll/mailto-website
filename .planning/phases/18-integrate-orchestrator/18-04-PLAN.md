<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| SPA shell → browser CSP engine | Meta-CSP policy delivered in page content |
| window.AI() → proxy or LM Studio | Endpoint selected by protocol, no auth token embedded |
| Shell origin → sandboxed iframe module | Browser-enforced sandbox + CSP frame-src 'self' |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-18-14 | Tampering | Generated module XSS | mitigate | CSP `script-src` whitelists only 'self' + pinned CDNs; `frame-src 'self'` prevents cross-origin iframe loading |
| T-18-15 | Information Disclosure | Authorization header in network logs | mitigate | Header removed from window.AI() fetch; proxy injects/strips auth server-side |
| T-18-16 | Spoofing | Module loads arbitrary CDN script | mitigate | CSP `script-src` explicitly lists jsdelivr + tailwindcss; any other CDN blocked by browser |
| T-18-17 | Tampering | `connect-src` allowing localhost | accept | Required for local development direct-to-LM-Studio path. HTTPS deployments don't need it (proxy is same-origin `'self'`) but allowing it is harmless in HTTPS context since browsers block mixed-content localhost anyway |
| T-18-18 | Elevation of Privilege | Meta CSP bypass via injected `<meta>` | accept | Only applies if attacker can inject HTML before the meta tag; shell.html is template-owned, not user-generated. Header-based CSP deferred (requires deployer cooperation per CONTEXT.md) |
</threat_model>
