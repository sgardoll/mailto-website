    Implement the public function:
    ```python
    def distill(normalized_input: dict[str, Any], lm_cfg) -> MechanicSpec | None:
        """Convert normalized_input to MechanicSpec. Returns None for informational emails (D-13)."""
        source_type = normalized_input.get("source_type", "text")
        system = _build_system_prompt(source_type)
        user = _build_user_prompt(normalized_input)
        raw = lm_studio.chat_json(lm_cfg, system=system, user=user, schema=DISTILL_SCHEMA)
        try:
            return _parse_and_validate(raw, normalized_input.get("source_url"))
        except (JsonSchemaError, PydanticValidationError, ValueError) as e:
            log.warning("DISTILL validation failed (attempt 1): %s", e)
            user_retry = user + f"\n\nPrevious attempt failed: {e}\nPlease fix and retry."
            raw2 = lm_studio.chat_json(lm_cfg, system=system, user=user_retry, schema=DISTILL_SCHEMA)
            try:
                return _parse_and_validate(raw2, normalized_input.get("source_url"))
            except (JsonSchemaError, PydanticValidationError, ValueError) as e2:
                log.error("DISTILL validation failed (attempt 2): %s", e2)
                raise DistillFailed(str(e2)) from e2
    ```

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| normalized_input → distill() | Untrusted email content crosses here (body originated from external email) |
| distill() → caller (MechanicSpec) | Trust boundary: raw body MUST NOT exit this module — PIPE-03 |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-16-02-01 | Tampering | LM Studio output | mitigate | Constrained decoding via json_schema response_format (D-02) + post-hoc jsonschema.validate() (D-03) + Pydantic model_validate (D-09/D-10). Three independent validation layers before spec leaves the module. |
| T-16-02-02 | Information Disclosure | Raw email body leaking past DISTILL | mitigate | MechanicSpec type has no body/raw/original field (structural enforcement); test `test_spec_has_no_raw_body_fields` asserts this at the type level. Body is only used transiently inside `_build_user_prompt` for the LM call and never returned. |
| T-16-02-03 | Denial of Service | Malformed LM output causing retry storms | mitigate | Exactly one retry per D-14; DistillFailed raised on second failure; orchestrator catches and surfaces to user. No unbounded retry loop. |
| T-16-02-04 | Spoofing | LM declares a kind not in MechanicKind enum | mitigate | Pydantic `kind: MechanicKind` enum coercion rejects unknown values (VALUES are validated by enum conversion); triggers retry then DistillFailed. |
</threat_model>
