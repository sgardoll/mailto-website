File structure (implement all sections):

SECTION 1 — Helpers (showError, clearError, attachValidation with touched flag):
- showError(inputId, message): sets span textContent, sets span.hidden=false, sets aria-invalid='true' on input. Use .textContent not .innerHTML.
- clearError(inputId): clears span textContent, sets span.hidden=true, sets aria-invalid='false' on input
- attachValidation(inputId, rule): attaches blur listener (sets touched=true, calls check()), attaches input listener (calls check() if touched). rule is function(input) returning error string or ''.

SECTION 2 — Validation rules (one function per field):
- ruleGmailAddress: empty or input.validity.typeMismatch -> "Enter a valid Gmail address"
- ruleAppPassword: value.trim().length < 16 -> "Enter your Gmail app password (16 characters)"
- ruleGmailFolder: empty -> "Folder name cannot be blank"
- ruleLmsBaseUrl: empty OR new URL(v) throws -> "Enter a valid URL (e.g. http://localhost:1234/v1)"
- ruleLmsModel: empty -> "Model tag is required"
- ruleLmsTemperature: input.validity.rangeUnderflow OR rangeOverflow -> "Temperature must be between 0.0 and 2.0"
- ruleLmsMaxTokens: input.validity.rangeUnderflow OR stepMismatch OR badInput -> "Max tokens must be a positive whole number"
- ruleLmsCliPath: empty -> "CLI path is required (default: lms)"

SECTION 3 — Wire validations:
Call attachValidation() for each of the 8 fields above.

SECTION 4 — Show/Hide toggle:
querySelectorAll('.toggle-visibility').forEach: on click, toggle input.type between 'password'/'text', update button .textContent ('Show'/'Hide'), update aria-label, update aria-pressed. SECURITY: never read or log the input value in this handler.

SECTION 5 — Gmail fan-out display:
getElementById('gmail-address').addEventListener('input'): update imap-user-preview.textContent and smtp-user-preview.textContent with this.value.

SECTION 6 — Allowed senders widget:
- updateRemoveButtons(): disable Remove button when sendersList has exactly 1 .sender-row
- attachSenderValidation(input): per-row touched+blur/input validation. Creates a .sender-error span inside the row using document.createElement (not innerHTML). Sets .textContent on the span.
- Initial row: wire validation on first .sender-input; call updateRemoveButtons()
- Add sender button: on click, createElement('div') for .sender-row, createElement('input') type email .sender-input, createElement('button') .remove-sender, append to sendersList, wire validation, updateRemoveButtons(), focus new input
- Remove button: on click (via event delegation on sendersList), remove .sender-row from DOM, updateRemoveButtons()

SECTION 7 — Form submit:
- form.addEventListener('submit', ...): e.preventDefault()
- Collect senders: Array.from(querySelectorAll('.sender-input')).map(function(i){return i.value.trim();}).filter(Boolean)
- Validate senders.length >= 1; if 0, show allowed-senders-error and return
- Build payload object (all keys listed in interfaces block)
- SECURITY: do NOT console.log payload — it contains the app password

  - on ok=true: clear form-error-summary
  - on ok=false: iterate data.errors, for each error find span by field name (replace _ with - for ID lookup), set span.textContent = err.message, span.hidden = false; show form-error-summary
  - on catch: show network error in form-error-summary using .textContent

SECTION 8 — Exit button:
- getElementById('exit-btn').addEventListener('click'): disable button, set textContent to 'Exiting...', fetch('/exit', POST), on ok=true set document.body.textContent (not innerHTML) or use a known safe element to display the shutdown message

Key implementation note for Exit: Since document.body.textContent would erase the whole page, prefer finding a known container and setting .textContent on it, or replacing with a newly created paragraph via document.createElement. Do NOT use .innerHTML with any string from server response.

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Browser DOM -> fetch body | User-entered values (including app password) are serialised to JSON and POSTed |
| fetch response -> DOM | Server error messages are written to error span elements |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-05-01 | Information Disclosure | gmail-app-password in browser | mitigate | NEVER call console.log() with payload or any password-field value. Password is read once into payload object, then JSON.stringify'd into fetch body. This is enforced as an acceptance criterion: grep for console.log must return empty. |
| T-02-05-02 | Tampering | Server error messages written to DOM | mitigate | Error messages must be set via .textContent (not .innerHTML) on error spans. This is enforced: executor MUST use .textContent for all DOM text mutations. |
| T-02-05-03 | Tampering | allowed_senders serialised as string | mitigate | Array.from().map().filter() pattern enforces JSON array type. Server validate() enforces isinstance(list) as a secondary check. |
| T-02-05-04 | Tampering | CSRF on POST /validate-form | accept | Localhost only, single-user, no auth cookies. Flask route uses request.get_json() which requires Content-Type: application/json — cross-origin form submissions cannot set this header without a preflight. |
</threat_model>
