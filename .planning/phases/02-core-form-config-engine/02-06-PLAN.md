<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| HTTP POST body -> Flask route | JSON body from browser enters server |
| Flask route -> _wizard_state | Validated form data is stored in module-level dict |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-06-01 | Tampering | POST /validate-form input | mitigate | `request.get_json(force=True)` parses JSON body. Builder's `validate()` checks required fields before `build()` is called. No raw user input reaches yaml.dump() without passing validate() first. |
| T-02-06-02 | Tampering | YAML injection via _wizard_state | mitigate | builder.build() uses yaml.dump() exclusively — never f-string YAML. PyYAML handles quoting of special characters in user-supplied strings. |
| T-02-06-03 | Information Disclosure | app password in _wizard_state | accept | _wizard_state is a server-side in-memory Python dict, never serialised to disk or logged. It holds the app password only for the duration of the session, consistent with how .env will store it in Phase 4. |
| T-02-06-04 | Tampering | CSRF on POST /validate-form | accept | Localhost only. Single-user local tool. No auth cookies. request.get_json(force=True) requires Content-Type: application/json which cross-origin form submissions cannot set without a CORS preflight — natural barrier. Document this rationale explicitly. |
</threat_model>
